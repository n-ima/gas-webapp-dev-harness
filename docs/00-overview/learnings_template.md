# 教訓ログ（learnings）

このプロジェクトの進行中に得られた「次から同じ失敗をしないための教訓」を1行ずつ追記する。
このファイルはセッション開始時にフックで自動的にエージェントへ注入されるため、
ここに書いたことは以後のすべてのセッションで前提として扱われる。

## 記録するタイミング（エージェント向けルール）

- ユーザーに同じ趣旨の訂正を受けたとき（1回でも記録してよい。2回目なら必ず記録する）。
- 同じアプローチで2回失敗したとき。
- プロジェクト固有の暗黙知（このリポジトリではXを使う、Yはしない等）が判明したとき。
- **コマンド・ツールの実行方法を試行錯誤の末に確立したとき**（成功したコマンドを
  そのまま記録する。実行方法の獲得知識は書かないと消え、次のセッションが
  同じ試行錯誤を繰り返す最大の原因になる。仕様の教訓より優先して書く）。
- セッションを終える・別フェーズへ引き継ぐ前に、このセッションで確立した
  環境固有の実行方法が記録済みか確認する（未記録のまま次へ渡さない）。

書式: `- [YYYY-MM-DD] <教訓を1行で。理由が自明でなければ括弧で補足>`
環境系の例: `- [YYYY-MM-DD] デプロイはCLIがPATHに無いため python -m <tool> で実行する`

記録しないもの: 一般的なベストプラクティス（それはスキルに書く）、
一時的な作業メモ（それは各フェーズのドキュメントに書く）。
