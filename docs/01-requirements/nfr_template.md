# 非機能要件（NFR）

| 区分 | 要件 | 目標値・基準 | 備考 |
|---|---|---|---|
| 性能 | レスポンスタイム | | |
| 性能 | 同時アクセス数 | | |
| 可用性 | 稼働率目標 | | |
| 可用性 | 障害時の復旧目標(RTO/RPO) | | |
| セキュリティ | 認証・認可方式 | | |
| セキュリティ | データ保護（暗号化等） | | |
| セキュリティ | OWASP Top10相当の考慮 | | |
| 運用・監視 | ログ・監視方針 | | |
| 運用・監視 | バックアップ方針 | | |
| スケーラビリティ | 想定データ量・成長見込み | | |
| 対応環境 | 対応OS/ブラウザ/デバイス | | |
| コスト | インフラ・運用コスト上限 | | |
| コンプライアンス | 法規制・社内ポリシー | | |
| アクセシビリティ | 対応レベル（該当する場合） | | |
| 国際化 | 多言語対応の要否 | | |

必要に応じて行を追加・削除してよい。「該当なし」も明示的に記載する（空欄のまま残さない）。
