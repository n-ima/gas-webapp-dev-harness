## 自動化できる/できない境界

エージェントが実際にリリースを自動実行するにあたり、**どこまでを自動化してよいか**を明確にする。

| 作業 | 自動化可否 | 理由・人手が必要な場合の担当 |
|---|---|---|
| ビルド・パッケージング | 自動 | |
| テスト実行 | 自動 | |
| デプロイ（インフラへの反映） | 自動 / 人手 | 例: 支払い情報の入力、外部ダッシュボードでの手動承認、ドメイン購入は人手 |
| DBマイグレーション | 自動 / 人手 | |
| 監視・アラート設定 | 自動 / 人手 | |
| 本番切り替え（カットオーバー） | 自動 / 人手 | |

「人手」に分類した項目は、リリースエージェントが該当ステップで作業内容を提示し、
ユーザーに実行してもらってから先に進む。
